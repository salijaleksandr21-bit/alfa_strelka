# 🤖 Alfa Strelka: Автономный Генератор Приложений
## «Дошутились»
**Alfa Strelka** — это система на основе LLM, которая автономно генерирует готовое веб-приложение и полный комплект проектной документации на основе бизнес-требований и описания бизнес-процесса.

Решение для хакатона **«Автономная команда разработки»**, где главная задача — построить генератор, способный создавать другие приложения от идеи до кода.

## 🚀 Ключевые возможности

- **Полная автономность** — система работает без вмешательства человека.
- **Каскадная генерация** — строгое следование цепочке:  
  БТ/БП → Use Cases → НФТ → ФТ → Исходный код → Тесты → README.
- **Трассируемость** — каждый артефакт (например, функциональное требование) содержит ссылки на исходные бизнес-требования.
- **Экспертная самопроверка** — встроенный агент-эксперт анализирует результаты и при необходимости отправляет компоненты на доработку.  
  Предусмотрено три режима строгости: `Lite`, `Default`, `Strict`.
- **Режим доработки** — возможность точечно изменить результат через короткий комментарий к уже сгенерированному проекту.

## 🛠️ Технологический стек

- **Язык**: Python 3.11+
- **LLM-провайдер**: OpenRouter (унифицированный доступ к LLM)
- **Модель**: рекомендуется `google/gemma-2-9b-it` (экономичная, <$1 за 1M выходных токенов)
- **Стек генерируемого приложения**: единый для всех заданий — HTML, CSS, Vanilla JS (запуск в браузере без сборки)

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/salijaleksandr21-bit/alfa_strelka.git
cd alfa_strelka
pip install -r requirements.txt
```

### 2. Настройка API-ключа OpenRouter

1. Зарегистрируйтесь на [OpenRouter.ai](https://openrouter.ai).
2. В разделе **Keys** создайте новый ключ.
3. Скопируйте ключ (начинается с `sk-or-v1-...`).
4. В корне проекта создайте файл `.env` и добавьте:

```ini
OPENROUTER_API_KEY=ваш_ключ
```

*Файл `.env` уже в `.gitignore` – ключ не попадёт в открытый доступ.*

### 3.1 Запуск генерации (Консоль)

Пример для задания **B (Таск-трекер)**:

```bash
python console.py --input examples/B --output ./my_awesome_app
```

В консоли отображаются шаги, использованные токены и вердикты эксперта.  
Готовое приложение и документация появятся в папке `./my_awesome_app`.

### 3.2 Запуск генерации (Сайта)

```bash
python app.py
```

Всё готово. Страница для генерирования доступна по адресу http://127.0.0.1:5000/
В публичный доступ выставлена страница https://technostrelka2026.pythonanywhere.com/

## ⚙️ Конфигурация и режимы

Все настройки находятся в `config/settings.py`. Их можно переопределить через `.env`:

```ini
OPENROUTER_API_KEY=...    # API-ключ
DEFAULT_MODEL=...         # модель по умолчанию 
EXPERT_LEVEL=default      # strict / default / lite
MAX_RETRIES=3             # число повторных попыток при сбое
```

**Режимы эксперта** (`EXPERT_LEVEL`):

- `lite` – мягкий контроль, быстрый прототип
- `default` – сбалансированная строгость
- `strict` – жёсткое соответствие требованиям



## 📥 Входные данные (папка `--input`)

Ожидаются Markdown-файлы внутри input папки:

- `БТ.md` – бизнес-требования (обязательно)
- `БП.md` – бизнес-процесс (обязательно)
- `Features.md` – произвольные характеристики (опционально)

Примеры input: папки `examples/A`, `examples/B`, `examples/C` и `examples/D`.
Примеры output: папки `examples/outputA`, `examples/outputB`, `examples/outputC`.

## 🤖 Выбор LLM для генерации кода

Проект использует **OpenRouter** для доступа к различным языковым моделям. Вы можете легко переключаться между моделями, меняя параметр `DEFAULT_MODEL` в файле `config/settings.py`.

### Какие модели мы протестировали

| Модель | Основные характеристики | Рекомендация |
|--------|------------------------|---------------|
| `deepseek-v4-flash` | 284B MoE, 1M контекст, очень дёшево (MIT) | **По умолчанию, но не оптимален** – мощный, но менее стабилен для сложных каскадных сценариев. |
| `qwen3.5_flash` | 35B total (3B active), отличный код и агентные задачи, 256K контекста | **Рекомендуется для разработки** – стабильный, дешёвый, надёжный фундамент. |
| `gemma-4-31b-it` | 31B dense, отличный баланс качества/цены, Apache 2.0 | **Рекомендуется для финальной генерации** – выдает более чистые веб‑приложения. |
| `llama-3.3-70b-instruct` | 70B dense, высокое качество рассуждений, но дороже | **Хорош для тестов** – другой стиль генерации, можно использовать как референс. |

### Как сменить модель

1. Откройте `config/settings.py`
2. Найдите строку:
`DEFAULT_MODEL: str = os.getenv("DEFAULT_MODEL", "qwen/qwen3.5-flash-02-23")`
3. Замените второй аргумент на желаемый идентификатор модели, например:
`DEFAULT_MODEL: str = os.getenv("DEFAULT_MODEL", "google/gemma-4-31b-it")`
5. Сохраните файл и перезапустите оркестратор.
