## Нефункциональные Требования (НФТ)
### Безопасность
* **НФТ-01**: Все пользовательские вводы должны проходить валидацию и санитизацию для предотвращения XSS/инъекций, чтобы предотвратить атаки на систему и пользовательские данные.
* **НФТ-02**: Система должна использовать защищённое соединение (HTTPS) для передачи данных между клиентом и сервером, чтобы предотвратить перехват и подмену данных.
* **НФТ-03**: Система не должна раскрывать внутренние API или учётные данные в клиентском коде, чтобы предотвратить несанкционированный доступ к системе.

### Конфиденциальность
* **НФТ-04**: Личные данные не покидают браузер пользователя; любое локальное хранилище должно быть зашифровано, чтобы обеспечить конфиденциальность пользовательских данных.
* **НФТ-05**: Система должна собирать и хранить только те данные, которые необходимы для функционирования, и удалять их после окончания сеанса, чтобы минимизировать риск утечки данных.

### Производительность
* **НФТ-06**: Система должна быть способна обрабатывать не менее 100 запросов в минуту без значительного снижения производительности, чтобы обеспечить быстрый и эффективный обмен данными.
* **НФТ-07**: Система должна быть оптимизирована для работы на устройствах с низкими характеристиками, чтобы обеспечить доступность для пользователей с различными устройствами.

### Удобство использования
* **НФТ-08**: Система должна иметь интуитивно понятный и простой интерфейс, чтобы пользователи могли легко ориентироваться и использовать систему без дополнительной помощи.
* **НФТ-09**: Система должна предоставлять rõкие и понятные сообщения об ошибках и инструкции, чтобы помочь пользователям решить проблемы и продолжить работу.

### Надёжность
* **НФТ-10**: Система должна быть способна восстанавливаться после сбоев и ошибок без потери данных, чтобы обеспечить непрерывность работы и минимизировать риск потери данных.
* **НФТ-11**: Система должна иметь механизмы резервного копирования и восстановления данных, чтобы обеспечить сохранность данных в случае сбоя или катастрофы.

Примечание: В бизнес-требованиях и бизнес-процессе не содержится небезопасных предложений, поэтому не требуется замена или добавление каких-либо требований для обеспечения безопасности.
