## Функциональные Требования (ФТ)

### ФТ-01: Выбор валюты
* **ID:** ФТ-01
* **Название:** Выбор исходной и целевой валюты
* **Описание:** Система должна предоставить пользователю возможность выбрать исходную и целевую валюту из списка доступных валют.
* **Входные данные:** Пользователь выбирает исходную и целевую валюту из выпадающего списка.
* **Ожидаемый результат:** Система отображает выбранные валюты.
* **Примечания по безопасности:** Система должна валидировать и санитизировать пользовательский ввод для предотвращения XSS-атак.

### ФТ-02: Отображение курса обмена
* **ID:** ФТ-02
* **Название:** Отображение актуального курса обмена
* **Описание:** Система должна отображать актуальный курс обмена, полученный из внешнего API.
* **Входные данные:** Система загружает актуальные курсы из API.
* **Ожидаемый результат:** Система отображает актуальный курс обмена.
* **Примечания по безопасности:** Система должна проверять ответ API на корректность и безопасность перед отображением его пользователю.

### ФТ-03: Конвертация валюты
* **ID:** ФТ-03
* **Название:** Конвертация валюты
* **Описание:** Система должна конвертировать введенную пользователем сумму в соответствии с выбранными валютами и отображать результат.
* **Входные данные:** Пользователь вводит сумму и выбирает исходную и целевую валюту.
* **Ожидаемый результат:** Система отображает сконвертированную сумму и курс.
* **Примечания по безопасности:** Система должна валидировать и санитизировать пользовательский ввод для предотвращения XSS-атак и проверять корректность введенной суммы.

### ФТ-04: Смена направления
* **ID:** ФТ-04
* **Название:** Смена направления конвертации
* **Описание:** Система должна поменять местами исходную и целевую валюты и пересчитать результат конвертации при нажатии кнопки «⇄».
* **Входные данные:** Пользователь нажимает кнопку «⇄».
* **Ожидаемый результат:** Система меняет местами исходную и целевую валюты и отображает обновленный результат конвертации.
* **Примечания по безопасности:** Система должна проверять корректность введенной суммы и выбранных валют перед пересчетом результата конвертации.

### ФТ-05: Обработка недоступности API
* **ID:** ФТ-05
* **Название:** Обработка недоступности API
* **Описание:** Система должна использовать последний закешированный курс и отображать его с пометкой о дате, если API не отвечает.
* **Входные данные:** API не отвечает.
* **Ожидаемый результат:** Система отображает последний закешированный курс с пометкой о дате.
* **Примечания по безопасности:** Система должна проверять корректность последнего закешированного курса перед отображением его пользователю.

### ФТ-06: Обработка некорректной суммы
* **ID:** ФТ-06
* **Название:** Обработка некорректной суммы
* **Описание:** Система должна показывать сообщение «Введите корректную сумму» и не выполнять конвертацию, если пользователь вводит отрицательное число или нечисловое значение.
* **Входные данные:** Пользователь вводит некорректную сумму.
* **Ожидаемый результат:** Система показывает сообщение «Введите корректную сумму» и не выполняет конвертацию.
* **Примечания по безопасности:** Система должна валидировать и санитизировать пользовательский ввод для предотвращения XSS-атак.

### ФТ-07: Доступность через веб-браузер
* **ID:** ФТ-07
* **Название:** Доступность через веб-браузер
* **Описание:** Система должна быть доступна через веб-браузер.
* **Входные данные:** Пользователь открывает веб-браузер и вводит адрес конвертера валют.
* **Ожидаемый результат:** Система отображает страницу конвертера валют.
* **Примечания по безопасности:** Система должна использовать HTTPS для шифрования данных и проверять корректность введенного адреса.

### ФТ-08: Валидация и санитизация пользовательского ввода
* **ID:** ФТ-08
* **Название:** Валидация и санитизация пользовательского ввода
* **Описание:** Система должна валидировать и санитизировать пользовательский ввод для предотвращения XSS-атак и других уязвимостей.
* **Входные данные:** Пользователь вводит данные.
* **Ожидаемый результат:** Система проверяет корректность и безопасность введенных данных.
* **Примечания по безопасности:** Система должна использовать проверенные библиотеки и методы для валидации и санитизации пользовательского ввода.

### ФТ-09: Шифрование данных
* **ID:** ФТ-09
* **Название:** Шифрование данных
* **Описание:** Система должна шифровать данные, передаваемые между клиентом и сервером.
* **Входные данные:** Пользователь вводит данные.
* **Ожидаемый результат:** Система шифрует данные перед передачей.
* **Примечания по безопасности:** Система должна использовать HTTPS для шифрования данных.

### ФТ-10: Обработка ошибок API
* **ID:** ФТ-10
* **Название:** Обработка ошибок API
* **Описание:** Система должна обрабатывать ошибки API и отображать пользователю соответствующее сообщение.
* **Входные данные:** API возвращает ошибку.
* **Ожидаемый результат:** Система отображает сообщение об ошибке.
* **Примечания по безопасности:** Система должна проверять корректность ответа API перед обработкой ошибки.
