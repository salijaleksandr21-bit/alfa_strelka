Название: Управление задачами с учётом погоды

Актор: Зарегистрированный пользователь

Основной поток — регистрация и вход:
1. Пользователь открывает страницу приложения.
2. Система предлагает форму входа или регистрации.
3. Пользователь регистрируется (указывает логин, пароль, город).
4. Система сохраняет учётные данные в localStorage.
5. Пользователь входит с логином и паролем.
6. Система отображает дашборд и список задач, назначенных на пользователя.

Основной поток — создание задачи:
1. Пользователь нажимает «Создать задачу».
2. Появляется форма: название, описание, дедлайн, приоритет, исполнитель (выпадающий список из зарегистрированных пользователей).
3. Пользователь заполняет поля и нажимает «Сохранить».
4. Система валидирует: название не пустое, дедлайн не в прошлом, исполнитель существует.
5. Задача сохраняется со статусом «Открыта» и отображается в списке.

Основной поток — смена статуса:
1. Пользователь нажимает кнопку «Изменить статус» на карточке задачи (только если он исполнитель или создатель).
2. Выбирает новый статус из выпадающего списка.
3. Система обновляет статус и перерисовывает список.

Основной поток — просмотр дашборда:
1. Пользователь переходит на вкладку «Дашборд».
2. Система показывает:
   - количество задач по статусам (круговой график или просто цифры),
   - список просроченных задач (если есть),
   - для каждой задачи из списка исполнителя отображается иконка погоды (на основе API).

Альтернативный поток — недоступность API погоды:
*. Если API погоды не отвечает или город не указан, отображается стандартная иконка погоды «?» с подписью «нет данных».

Альтернативный поток — назначение задачи самому себе:
*. Пользователь может назначить задачу себе.

Альтернативный поток — удаление задачи:
*. Пользователь (только создатель) может удалить задачу. Система запрашивает подтверждение.
