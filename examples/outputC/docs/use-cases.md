# Юз-кейсы: Конвертация валюты

## UC-01: Конвертация валюты (основной и альтернативные потоки)

**ID:** UC-01  
**Актор:** Пользователь  
**Предусловия:**  
- Пользователь имеет доступ к веб-интерфейсу через браузер (БТ-07).  
- Система настроена на получение курсов из внешнего API и кеширование.  

**Результат:** Пользователь видит корректную сконвертированную сумму и актуальный курс.

### Основной поток (успешная конвертация)

1. Пользователь открывает страницу конвертера.  
2. Система загружает актуальные курсы из внешнего API и сохраняет их в кеше (БТ-02).  
3. Пользователь выбирает исходную валюту из выпадающего списка, содержащего минимум 10 популярных валют (БТ-05).  
4. Пользователь выбирает целевую валюту из того же списка (БТ-01).  
5. Пользователь вводит сумму (числовое положительное значение) в соответствующее поле (БТ-03).  
6. Система в реальном времени отображает сконвертированную сумму и текущий курс (БТ-03).  

**Безопасность:**  
- Ввод суммы проверяется на корректность (только числовые значения, отсутствие отрицательных чисел) (БТ-03, альтернативный поток 5a).  
- Система не передаёт личные данные пользователя; все операции выполняются на стороне клиента (валидация ввода).  

**Ссылки на БТ:** БТ-01, БТ-02, БТ-03, БТ-05, БТ-07.

---

### Альтернативный поток 1 — смена направления

**А-1:** Пользователь нажимает кнопку «⇄».  
- Исходная и целевая валюты меняются местами (БТ-04).  
- Сумма остаётся прежней, конвертация пересчитывается с новым направлением.  
- Результат отображается мгновенно.  

**Ссылка на БТ:** БТ-04.

---

### Альтернативный поток 2 — недоступность API

**А-2:** Пункт 2 основного потока. Если внешний API не отвечает:  
- Система использует последний закешированный курс (БТ-06).  
- Рядом с отображаемым курсом появляется пометка:  
  > «Курс от [дата последнего кеширования]. Актуальные данные временно недоступны».  
- Конвертация выполняется с этим курсом.  

**Безопасность:**  
- Кеш хранится только в безопасном хранилище на сервере, без доступа к личным данным.  
- Дата кеширования не содержит персональной информации.  

**Ссылка на БТ:** БТ-06.

---

### Альтернативный поток 3 — некорректная сумма

**А-3:** Пункт 5 основного потока. Если пользователь вводит отрицательное число или нечисловое значение:  
- Система показывает сообщение «Введите корректную сумму» (БТ-03).  
- Конвертация не выполняется, результат не отображается.  
- Пользователь может исправить ввод.  

**Безопасность:**  
- Валидация на стороне клиента предотвращает передачу потенциально опасных данных на сервер.  

**Ссылка на БТ:** БТ-03.

---

## Примечания по безопасности

- Все внешние вызовы API выполняются с минимальными привилегиями (нет передачи секретов или учётных данных пользователя).  
- Кеширование не хранит персональные данные, только технические курсы и даты.  
- Интерфейс не позволяет изменять настройки безопасности или получать несанкционированный доступ.  
- Все входные данные строго валидируются (числовой формат, диапазон).  

Данные юз-кейсы полностью соответствуют бизнес-требованиям и правилам безопасности.
