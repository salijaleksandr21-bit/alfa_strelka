# Функциональные требования (ФТ) к конвертеру валют «CurrFlow»

## ФТ-01: Загрузка актуальных курсов из внешнего API

| Поле | Значение |
|------|----------|
| **ID** | ФТ-01 |
| **Источники** | БТ-02, UC-01 (шаг 2), Features (обновление каждые 60 с) |
| **Описание** | При открытии страницы и каждые 60 секунд система отправляет HTTP‑запрос к доверенному внешнему API для получения актуальных курсов обмена. Адрес API задаётся в конфигурации приложения (например, `https://api.exchangerate-api.com/v4/latest/USD`). |
| **Входные данные** | URL API (заранее определён) |
| **Ожидаемый результат** | Система получает ответ в формате JSON, содержащий курсы для минимум 11 валют (USD, EUR, RUB, GBP, JPY, CNY, TRY, KZT, BYN, GEL, UAH). Успешные данные сохраняются для последующего использования (см. ФТ-02). |
| **Примечания по безопасности** | - Валидировать ответ API: проверять, что тело ответа является валидным JSON с ожидаемой структурой (наличие ключей `rates`, `base`, `date`).<br>- Если в ответе содержатся строки (например, наименования валют), они должны быть экранированы при отображении для предотвращения XSS.<br>- Использовать HTTPS для защиты данных в пути.<br>- Обрабатывать ошибки сети (таймаут, 4xx, 5xx) и переход к кешу (см. ФТ-08). |

## ФТ-02: Кеширование курсов в localStorage

| Поле | Значение |
|------|----------|
| **ID** | ФТ-02 |
| **Источники** | БТ-06, UC-01 (A-2), Features (жёлтая плашка) |
| **Описание** | После успешного получения курсов из API система сохраняет их в `localStorage` вместе с меткой времени последнего успешного обновления. При недоступности API система использует сохранённые курсы. |
| **Входные данные** | Объект с курсами (из ФТ-01) |
| **Ожидаемый результат** | - При успешном запросе: в `localStorage` записываются данные: `{ rates: {…}, updated_at: "2024-01-01T12:00:00Z" }`.<br>- При неудачном запросе: система считывает данные из `localStorage` и использует их для конвертации. |
| **Примечания по безопасности** | - В `localStorage` не сохраняется чувствительная информация (пароли, токены).<br>- При восстановлении из `localStorage` проверять, что данные имеют ожидаемый формат (объект с ключами `rates` и `updated_at`). Если формат некорректен, игнорировать и показывать ошибку.<br>- При отображении даты и курса из кеша экранировать значения (через `textContent` или безопасные шаблоны). |

## ФТ-03: Выбор исходной и целевой валюты из выпадающего списка

| Поле | Значение |
|------|----------|
| **ID** | ФТ-03 |
| **Источники** | БТ-01, БТ-05, UC-01 (шаги 3-4), Features (11 валют) |
| **Описание** | На странице отображаются два выпадающих списка: «Исходная валюта» и «Целевая валюта». Каждый содержит 11 значений: USD, EUR, RUB, GBP, JPY, CNY, TRY, KZT, BYN, GEL, UAH. Значения по умолчанию: USD (исходная) и RUB (целевая). При изменении выбора система обновляет результат конвертации (см. ФТ-05). |
| **Входные данные** | Действие пользователя – изменение выбранного пункта в выпадающем списке. |
| **Ожидаемый результат** | Новый выбор мгновенно становится активным, система пересчитывает результат. |
| **Примечания по безопасности** | - Список формируется статически, без пользовательского ввода – уязвимости XSS нет.<br>- При восстановлении из `localStorage` (ФТ-09) проверять, что сохранённое значение есть в списке, иначе использовать значение по умолчанию. |

## ФТ-04: Ввод суммы с валидацией на лету

| Поле | Значение |
|------|----------|
| **ID** | ФТ-04 |
| **Источники** | БТ-03, UC-01 (шаг 5, A-3), Features (только числа с точкой) |
| **Описание** | Поле ввода суммы принимает только цифры (0-9) и одиночную точку `.` в качестве десятичного разделителя. Ввод символов вне этого набора (буквы, минус, запятая) блокируется нажатием клавиш или очищается. Дополнительно проверяется, что значение является неотрицательным числом (ноль допускается, но конвертация не изменит результат). |
| **Входные данные** | Пользовательский ввод в текстовое поле. |
| **Ожидаемый результат** | - При корректном вводе (например, `123.45`) система отображает результат конвертации (ФТ-05).<br>- При вводе некорректного значения (отрицательное, несколько точек, буквы) система не разрешает ввод недопустимого символа (символ не отображается) и / или показывает сообщение «Введите корректную сумму» под полем. Конвертация не выполняется. |
| **Примечания по безопасности** | - **Валидация**: проверка ввода на соответствие шаблону `^[0-9]*\.?[0-9]*$` и отсутствие отрицательного знака.<br>- **Экранирование вывода**: при отображении значений суммы в поле ввода и результатах – использовать `textContent` или экранирование HTML‑сущностей, чтобы предотвратить XSS при возможном внедрении через localStorage (см. ФТ-09).<br>- Санитизация для чисел не требуется, кроме проверки формата. |

## ФТ-05: Мгновенное отображение результата конвертации и текущего курса

| Поле | Значение |
|------|----------|
| **ID** | ФТ-05 |
| **Источники** | БТ-03, UC-01 (шаг 6), Features (округление 2 знака, показывать курс) |
| **Описание** | При каждом изменении исходной/целевой валюты или суммы система в реальном времени вычисляет конвертированную сумму как `сумма * курс_исходной_к_целевой` и отображает результат, округлённый до двух знаков после запятой. Также отображается строка вида «1 USD = 94.24 RUB» (для примера). |
| **Входные данные** | Текущие значения: исходная валюта, целевая валюта, сумма. |
| **Ожидаемый результат** | - Результат конвертации появляется рядом с полем суммы или в специальной области.<br>- Курс обновляется синхронно.<br>- Если ни одна валюта не выбрана или сумма не введена – результат не отображается или показываются прочерки. |
| **Примечания по безопасности** | - При отображении названий валют (строки из API) – экранировать их.<br>- Числовые значения экранировать не требуется, но для единообразия следует использовать безопасные методы вставки (например, `textContent`). |

## ФТ-06: Кнопка смены валют местами с автоматическим пересчётом

| Поле | Значение |
|------|----------|
| **ID** | ФТ-06 |
| **Источники** | БТ-04, UC-01 (A-1), Features (кнопка «⇄» пересчитывает) |
| **Описание** | На странице размещена кнопка «⇄». При её нажатии значения выбранных исходной и целевой валют меняются местами. Сумма остаётся неизменной, а результат конвертации мгновенно пересчитывается для нового направления. Дополнительного нажатия не требуется. |
| **Входные данные** | Клик по кнопке. |
| **Ожидаемый результат** | Исходная валюта становится целевой, целевая – исходной. Курс меняется на обратный (если не задан прямой курс, система может вычислить его как `1 / исходный_курс`). Результат конвертации обновляется. |
| **Примечания по безопасности** | Нет дополнительных требований к безопасности. |

В верхней части страницы появляется жёлтая плашка (баннер) с текстом «Данные устарели».<br>При последующем успешном обновлении API и текст, и плашка исчезают. |
| **Входные данные** | Состояние ошибки запроса к API. |
| **Ожидаемый результат** | См. описание. |

## ФТ-09: Сохранение и восстановление состояния в localStorage

| Поле | Значение |
|------|----------|
| **ID** | ФТ-09 |
| **Источники** | Features (сохранять последние выбранные валюты и сумму) |
| **Описание** | При каждом изменении исходной валюты, целевой валюты или суммы система записывает эти значения в `localStorage` под ключами `sourceCurrency`, `targetCurrency`, `amount`. При открытии страницы система проверяет наличие сохранённых данных. Если они существуют и валидны, поля восстанавливаются из хранилища, и конвертация выполняется автоматически. Если данных нет или они повреждены – используются значения по умолчанию (USD, RUB, сумма 1). |
| **Входные данные** | События изменения полей (change / input). |
| **Ожидаемый результат** | - Сохранение: при каждом изменении данные пишутся в `localStorage`.<br>- Восстановление: при загрузке страницы поля заполняются сохранёнными значениями. |
| **Примечания по безопасности** | - В `localStorage` хранятся только неконфиденциальные данные.<br> - **Валидация при восстановлении**:<br>   - `sourceCurrency` и `targetCurrency` должны быть строками, присутствующими в списке из 11 валют. Иначе – использовать значения по умолчанию.<br>   - `amount` должен быть строкой, соответствующей числу с точкой (проверка регулярным выражением `^[0-9]*\.?[0-9]+$`). Если нет – игнорировать и не восстанавливать.<br> - При вставке восстановленных значений в HTML использовать `textContent` или безопасные атрибуты. |
