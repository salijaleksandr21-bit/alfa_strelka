## Нефункциональные требования (НФТ)

| Категория | ID | Описание |
|-----------|----|----------|
| **Безопасность** | НФТ‑01 | Все пользовательские вводы (сумма, выбор валют) должны проходить строгую валидацию и санитизацию перед отображением или передачей на сервер для предотвращения XSS и инъекций. Некорректные значения (отрицательные числа, нечисловые символы) должны отвергаться на стороне клиента до отправки любого запроса. |
| **Конфиденциальность** | НФТ‑02 | История конвертации или введённые суммы не должны сохраняться или передаваться за пределы браузера пользователя. Если используется локальное хранилище (LocalStorage/IndexedDB) для кэширования курсов, все данные должны быть зашифрованы симметричным алгоритмом (например, AES‑256) с ключом, генерируемым на стороне клиента. |
| **Безопасные настройки по умолчанию** | НФТ‑03 | Ключи доступа к внешнему API (если используются) не должны присутствовать в клиентском коде в открытом виде. Все вызовы API должны выполняться через серверный прокси (BFF) или, в случае прямого вызова, с использованием переменных окружения на стороне сервера. Клиентский код не должен раскрывать внутренние URL, endpoints или учётные данные. |
| **Производительность** | НФТ‑04 | Результат конвертации должен вычисляться и отображаться не более чем за 200 мс после прекращения ввода (с учётом дебаунса 300 мс). При недоступности API (используется кэш) задержка не должна превышать 50 мс, так как все вычисления выполняются локально. |
| **Надёжность** | НФТ‑05 | Кэшированные курсы валют должны храниться с меткой времени и сохраняться в течение 24 часов (согласно требованию БТ-06). При обнаружении устаревшего кэша (старше 24 часов) система должна игнорировать его и отображать сообщение «Нет данных», запрашивая обновление вручную или через фоновую подгрузку при восстановлении сети. |

### Примечание
Исходные Features не содержали небезопасных предложений, поэтому все требования сформулированы без замен. Дополнительно добавлены требования производительности и надёжности, исходя из бизнес-процесса (мгновенная конвертация и корректная работа при сбоях API).
