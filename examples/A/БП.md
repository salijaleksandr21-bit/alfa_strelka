# Бизнес-процесс (БП)

Название: Управление задачами  
Актор: Пользователь

## Основной поток — создание задачи
1. Пользователь нажимает кнопку «Новая задача».
2. Система отображает форму с полями «Название» и «Описание».
3. Пользователь заполняет поля и нажимает «Создать».
4. Система валидирует: название не должно быть пустым.
5. Система создаёт задачу со статусом «К выполнению» и отображает её на доске.

## Основной поток — смена статуса
1. Пользователь перетаскивает карточку задачи в другую колонку (или нажимает кнопку смены статуса).
2. Система обновляет статус задачи и перемещает карточку.

## Основной поток — удаление
1. Пользователь нажимает кнопку удаления на карточке.
2. Система запрашивает подтверждение.
3. При подтверждении — задача удаляется.

## Альтернативный поток — пустое название
- 4a. Система показывает сообщение «Название задачи обязательно» и не создаёт задачу.

## Альтернативный поток — отмена удаления
- 2a. При запросе подтверждения пользователь нажимает «Отмена» — удаление отменяется, задача остаётся.

## Альтернативный поток — фильтрация по статусу (опционально)
- Пользователь выбирает статус из выпадающего списка (например, «Все», «К выполнению», «В работе», «Выполнено»). Система отображает только задачи выбранного статуса.
