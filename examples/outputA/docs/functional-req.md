Вот детальные функциональные требования (ФТ) для веб-калькулятора «QuickCalc», разработанные на основе предоставленных бизнес-требований, юз-кейсов, Features и НФТ. Каждое требование проверено на соответствие правилам безопасности и этики.

---

### ФТ-01: Выполнение базовых арифметических операций

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-01 |
| **Ссылка на источник** | БТ-01, ЮК-01, Feature «Поле ввода показывает предыдущее выражение мелким шрифтом, текущий ввод — крупным» |
| **Входные данные** | Пользовательский ввод через графический интерфейс (нажатие на кнопки цифр 0-9, десятичной точки, операций +, −, ×, ÷ и кнопки «=»). |
| **Ожидаемый результат** | Система корректно вычисляет результат арифметической операции (сложение, вычитание, умножение, деление) и отображает его на экране. Поле ввода показывает предыдущее выражение (например, «2 + 3») мелким шрифтом, а текущий результат (например, «5») — крупным шрифтом. |
| **Описание** | Пользователь вводит первое число, выбирает операцию, вводит второе число и нажимает «=». Система выполняет вычисление на стороне клиента (JavaScript) и отображает результат. Ввод осуществляется исключительно через графический интерфейс (кнопки), что исключает прямую обработку строковых команд. |
| **Примечания по безопасности** | **Валидация ввода:** Все входные данные поступают только через нажатие кнопок (цифры, точка, операции). Система не принимает произвольный текстовый ввод с клавиатуры (если не предусмотрено отдельным требованием). **Санитизация:** Не требуется, так как ввод ограничен предопределёнными кнопками. **Кодирование вывода:** Результат вычисления (число) отображается как текст. Для защиты от XSS, если результат будет вставляться в DOM, необходимо использовать `textContent` вместо `innerHTML`. **Безопасность вычислений:** Все вычисления выполняются на стороне клиента. Не допускается использование `eval()` или `new Function()` для вычислений. Использовать безопасные методы, например, ручной парсинг и вычисление или библиотеку с изолированным контекстом. |

### ФТ-02: Обработка деления на ноль

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-02 |
| **Ссылка на источник** | БТ-04, ЮК-02, Feature «При делении на ноль показывать не только текст, но и красную рамку вокруг поля ввода» |
| **Входные данные** | Пользователь вводит первое число (ненулевое), выбирает операцию деления (÷), вводит второе число — 0, нажимает «=». |
| **Ожидаемый результат** | Система не выполняет вычисление. На экране отображается сообщение «Ошибка: деление на ноль». Вокруг поля ввода появляется красная рамка. Поле ввода остаётся неизменным (первое число, операция, второе число). |
| **Описание** | Система проверяет, что второе число равно нулю, а операция — деление. Если условие истинно, вычисление блокируется, и пользователю выводится понятное сообщение об ошибке с визуальным индикатором (красная рамка). |
| **Примечания по безопасности** | **Валидация ввода:** Проверка на ноль выполняется на стороне клиента перед любыми вычислениями. **Кодирование вывода:** Сообщение об ошибке отображается как текст. Использовать `textContent` для вставки в DOM. **Визуальный индикатор:** Красная рамка реализуется через CSS-класс, который добавляется к полю ввода. Не допускается вставка HTML-кода из сообщения об ошибке. |

### ФТ-03: Очистка текущего ввода (кнопка «C»)

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-03 |
| **Ссылка на источник** | БТ-03, ЮК-03, Feature «Кнопка «C» сбрасывает только текущий ввод, история не очищается» |
| **Входные данные** | Пользователь нажимает кнопку «C». |
| **Ожидаемый результат** | Поле ввода очищается, текущее число, операция и все временные данные сбрасываются. Экран отображает начальное состояние (например, «0»). История предыдущих вычислений (если есть) не удаляется. |
| **Описание** | Кнопка «C» предназначена для сброса текущего состояния калькулятора (введённые числа, выбранная операция, результат ошибки). История вычислений, сохранённая в localStorage, остаётся нетронутой. |
| **Примечания по безопасности** | **Валидация ввода:** Не требуется, так как действие инициируется нажатием кнопки. **Санитизация:** Не требуется. **Безопасность данных:** При сбросе состояния не происходит удаление данных из localStorage. Убедиться, что операция сброса не приводит к утечке временных данных (например, через кэш браузера). |

### ФТ-04: Просмотр истории вычислений (с сохранением в localStorage)

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-04 |
| **Ссылка на источник** | БТ-02 (расширенное), ЮК-04, Feature «История вычислений сохраняется в localStorage и доступна после перезагрузки страницы» |
| **Входные данные** | Пользователь нажимает кнопку/переключатель «История». |
| **Ожидаемый результат** | Система отображает список последних 10 вычислений в формате `2 + 3 = 5`. История сохраняется в localStorage браузера и доступна после перезагрузки страницы. Пользователь может нажать на запись истории, чтобы подставить её в текущее поле ввода. |
| **Описание** | После каждого успешного вычисления (ФТ-01) система сохраняет запись в локальном хранилище. При открытии панели истории отображаются последние 10 записей. При нажатии на запись, выражение подставляется в поле ввода для повторного использования. |
| **Примечания по безопасности** | **Валидация ввода:** Записи истории создаются только системой на основе выполненных вычислений. Пользователь не может напрямую редактировать историю. **Кодирование вывода:** При отображении истории в DOM использовать `textContent` для вставки текста (например, «2 + 3 = 5»), чтобы предотвратить XSS. **Безопасное хранение:** История не содержит чувствительных данных (паролей, токенов). Если в будущем планируется хранить что-то чувствительное, данные должны быть зашифрованы. **Локальное хранение:** Данные хранятся только в localStorage браузера пользователя. Не передаются на сервер. **Очистка истории:** Для очистки истории предусмотрена отдельная кнопка «Очистить историю» (см. ФТ-05). |

### ФТ-05: Очистка истории вычислений

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-05 |
| **Ссылка на источник** | Feature «Для очистки истории – отдельная кнопка «Очистить историю»» |
| **Входные данные** | Пользователь нажимает кнопку «Очистить историю». |
| **Ожидаемый результат** | Все записи истории вычислений удаляются из localStorage. Панель истории (если открыта) отображает пустой список или сообщение «История пуста». Текущее состояние калькулятора не изменяется. |
| **Описание** | Кнопка «Очистить историю» предназначена для полного удаления всех сохранённых записей истории. Это действие необратимо. |
| **Примечания по безопасности** | **Валидация ввода:** Не требуется, так как действие инициируется нажатием кнопки. **Санитизация:** Не требуется. **Безопасность данных:** Перед удалением данных из localStorage, рекомендуется запросить подтверждение у пользователя (например, через диалоговое окно `confirm()`), чтобы предотвратить случайное удаление. **Локальное хранение:** Операция выполняется только на стороне клиента. |

### ФТ-09: Защита от XSS при отображении данных

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-09 |
| **Ссылка на источник** | Все ФТ, где отображаются пользовательские данные (ФТ-01, ФТ-02, ФТ-04) |
| **Входные данные** | Любые данные, отображаемые в интерфейсе (результаты вычислений, сообщения об ошибках, записи истории). |
| **Ожидаемый результат** | Все данные, отображаемые в DOM, экранированы и не могут быть интерпретированы как HTML или JavaScript. |
| **Описание** | Система должна использовать безопасные методы вставки данных в DOM: `textContent` вместо `innerHTML`, или использовать современные фреймворки (React, Vue, Angular) с автоматическим экранированием. |
| **Примечания по безопасности** | **Кодирование вывода:** Это критическое требование для предотвращения XSS-атак. Все строки, которые могут содержать пользовательский ввод (даже если он вводится через кнопки), должны быть экранированы перед вставкой в HTML. **Проверка:** Провести статический анализ кода для выявления мест, где используется `innerHTML` или `document.write`. |

### ФТ-10: Защита от использования `eval()` и `new Function()`

| Элемент | Описание |
|---------|----------|
| **ID** | ФТ-10 |
| **Ссылка на источник** | ФТ-01 (выполнение вычислений) |
| **Входные данные** | Выражение, введённое пользователем (через кнопки). |
| **Ожидаемый результат** | Система выполняет вычисления без использования `eval()` или `new Function()`. |
| **Описание** | Для вычисления арифметических выражений необходимо использовать безопасные методы, такие как ручной парсинг и вычисление (например, с помощью стека или обратной польской записи) или использование специализированной библиотеки (например, `mathjs` с изолированным контекстом). |
| **Примечания по безопасности** | **Безопасность вычислений:** Использование `eval()` или `new Function()` позволяет выполнить произвольный JavaScript-код, что является критической уязвимостью. Данное требование явно запрещает их использование. **Альтернатива:** Если используется библиотека, убедиться, что она не имеет известных уязвимостей и не использует `eval()` внутри себя. |
