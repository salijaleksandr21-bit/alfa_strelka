Вот нефункциональные требования (НФТ) для веб-калькулятора, разработанные на основе предоставленных бизнес-требований, бизнес-процесса и с учётом правил безопасности и этики.

### Нефункциональные требования (НФТ)

| Категория | ID | Описание требования |
| :--- | :--- | :--- |
| **Безопасность** | НФТ-01 | **Валидация и санитизация ввода.** Все пользовательские вводы (нажатия кнопок, ввод с клавиатуры) должны проходить строгую валидацию и санитизацию на стороне клиента и сервера (если применимо) для предотвращения XSS-атак и инъекций. Допускаются только числовые символы, десятичный разделитель (точка) и символы операций (+, -, *, /). Любые другие символы должны игнорироваться или экранироваться. |
| **Конфиденциальность** | НФТ-02 | **Локальное хранение данных.** История вычислений (БТ-02) должна храниться исключительно в браузере пользователя (например, в `localStorage`). Любые данные, сохраняемые локально, должны быть зашифрованы с использованием стандартных криптографических API браузера (например, Web Crypto API). Система не должна передавать историю вычислений или любые другие данные на внешние серверы. |
| **Безопасность по умолчанию** | НФТ-03 | **Минимизация раскрытия информации.** Клиентский код (JavaScript, HTML, CSS) не должен содержать внутренних API-ключей, URL-адресов внутренних сервисов, учётных данных или комментариев, раскрывающих архитектуру системы. Все вычисления должны выполняться на стороне клиента. |
| **Производительность** | НФТ-04 | **Время отклика интерфейса.** Время между нажатием пользователем кнопки (цифры, операции, «=», «C») и отображением результата (или очисткой поля) не должно превышать 100 миллисекунд при стандартных условиях работы (современный браузер, средняя загрузка CPU). |
| **Надёжность** | НФТ-05 | **Обработка ошибок.** Система должна корректно обрабатывать все крайние случаи, включая: деление на ноль (БТ-04), переполнение (результат больше `Number.MAX_SAFE_INTEGER`), потерю точности при работе с дробными числами. В каждом случае пользователю должно отображаться понятное сообщение об ошибке (например, «Ошибка: деление на ноль», «Результат слишком велик»), а калькулятор должен оставаться в рабочем состоянии. |
| **Доступность** | НФТ-06 | **Доступность с клавиатуры.** Все функции калькулятора (ввод цифр, выбор операций, получение результата, сброс) должны быть доступны и управляемы исключительно с клавиатуры. Должна быть реализована поддержка клавиш: `0-9`, `.`, `+`, `-`, `*`, `/`, `Enter` (для «=»), `Escape` или `Backspace` (для «C»). |

**Примечание по безопасности:** В предоставленных Features не было небезопасных предложений, поэтому замена не потребовалась. Все требования сформулированы в соответствии с лучшими практиками безопасности.
