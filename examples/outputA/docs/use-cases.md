# Юз-кейсы для Веб-калькулятора

## ЮК-01: Выполнение базовой арифметической операции

| Элемент | Описание |
|---------|----------|
| **ID** | ЮК-01 |
| **Название** | Выполнение базовой арифметической операции |
| **Актор** | Пользователь |
| **Описание** | Пользователь вводит два числа, выбирает операцию и получает результат. |
| **Предварительные условия** | Веб-страница калькулятора загружена в браузере. Поле ввода пустое. |
| **Основной поток** | 1. Пользователь вводит первое число, нажимая цифровые кнопки. 2. Пользователь выбирает одну из операций: +, −, ×, ÷. 3. Пользователь вводит второе число. 4. Пользователь нажимает кнопку «=». 5. Система вычисляет результат и отображает его на экране. |
| **Альтернативные потоки** | — |
| **Постусловия** | Результат вычисления отображается на экране. Числа и операция сохраняются в контексте сессии для истории (если включена). |
| **Ссылки на БТ** | БТ-01, БТ-05 |

## ЮК-02: Обработка деления на ноль

| Элемент | Описание |
|---------|----------|
| **ID** | ЮК-02 |
| **Название** | Обработка деления на ноль |
| **Актор** | Пользователь |
| **Описание** | Пользователь пытается выполнить деление на ноль, система выводит понятное сообщение об ошибке. |
| **Предварительные условия** | Веб-страница калькулятора загружена. Пользователь выбрал операцию деления (÷). |
| **Основной поток** | 1. Пользователь вводит первое число (ненулевое). 2. Пользователь выбирает операцию деления (÷). 3. Пользователь вводит второе число — 0. 4. Пользователь нажимает «=». 5. Система определяет, что второе число равно нулю, и **не выполняет вычисление**. 6. Система отображает на экране сообщение: «Ошибка: деление на ноль». |
| **Альтернативные потоки** | — |
| **Постусловия** | Экран показывает сообщение об ошибке. Поле ввода остаётся неизменным (первое число, операция, второе число). Пользователь может продолжить ввод или нажать «C» для сброса. |
| **Ссылки на БТ** | БТ-04 |

## ЮК-03: Очистка ввода и сброс калькулятора

| Элемент | Описание |
|---------|----------|
| **ID** | ЮК-03 |
| **Название** | Очистка ввода и сброс калькулятора |
| **Актор** | Пользователь |
| **Описание** | Пользователь нажимает кнопку «C» для полной очистки текущего ввода и сброса состояния калькулятора. |
| **Предварительные условия** | Калькулятор находится в любом состоянии: ввод числа, выбор операции, отображение результата или ошибки. |
| **Основной поток** | 1. Пользователь нажимает кнопку «C». 2. Система очищает поле ввода, сбрасывает текущее число, операцию и все временные данные. 3. Экран отображает начальное состояние (обычно «0» или пустое поле). |
| **Альтернативные потоки** | — |
| **Постусловия** | Состояние калькулятора возвращено к начальному. История предыдущих вычислений (если есть) не удаляется. |
| **Ссылки на БТ** | БТ-03 |

## ЮК-04: Просмотр истории вычислений (опционально)

| Элемент | Описание |
|---------|----------|
| **ID** | ЮК-04 |
| **Название** | Просмотр истории последних вычислений |
| **Актор** | Пользователь |
| **Описание** | Пользователь просматривает список последних 10 выполненных вычислений в текущей сессии. |
| **Предварительные условия** | Выполнено хотя бы одно вычисление (ЮК-01). Функция истории включена. Пользователь не закрывал вкладку браузера (сессия активна). |
| **Основной поток** | 1. Пользователь нажимает кнопку или переключатель «История» (или аналогичный элемент интерфейса). 2. Система отображает список последних 10 вычислений в виде: `2 + 3 = 5` и т.д. 3. Пользователь просматривает записи. 4. Пользователь закрывает панель истории. |
| **Альтернативные потоки** | **4a.** Пользователь нажимает на одну из записей истории — система подставляет её в текущее поле ввода (для повторного использования). |
| **Постусловия** | Состояние калькулятора не меняется, если запись не была выбрана. Панель истории скрыта. |
| **Ссылки на БТ** | БТ-02, БТ-05 |

### Примечания по безопасности

Все сценарии реализуются в безопасном окружении:
- Ввод данных происходит только через графический интерфейс (цифровые кнопки), валидация выполняется на стороне клиента (нет прямой обработки строковых команд).
- Система не собирает, не хранит и не передаёт персональные данные пользователей.
- История вычислений хранится локально в рамках сессии браузера и не покидает устройство пользователя.
- Деление на ноль обрабатывается без исключительных ситуаций (например, без генерации ошибок на сервере, так как все вычисления выполняются на стороне клиента).
