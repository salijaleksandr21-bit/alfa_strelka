# QuickCalc — Веб-калькулятор

Простой, безопасный и удобный калькулятор для базовых арифметических операций. Работает полностью на стороне клиента, не требует установки и не отправляет данные на внешние серверы.

## Требования

- **Браузер**: Любой современный веб-браузер с поддержкой HTML5, CSS3 и JavaScript ES6+ (например, последние версии Chrome, Firefox, Safari, Edge).
- **Интернет**: Не требуется для работы приложения. История вычислений сохраняется локально в браузере.

## Установка

Установка не требуется. Приложение представляет собой набор статических файлов (HTML, CSS, JavaScript), которые запускаются непосредственно в браузере.

## Запуск приложения

1. Скачайте или скопируйте следующие файлы в одну папку:
   - `index.html`
   - `style.css`
   - `app.js`
2. Откройте файл `index.html` в любом современном браузере.

   **Примечание:** Для корректной работы убедитесь, что все три файла находятся в одной директории (структура папок не менялась).

3. Калькулятор готов к использованию. Для взаимодействия можно:
   - нажимать кнопки мышью,
   - использовать клавиатуру (цифры 0–9, точка, операции `+`, `-`, `*`, `/`, `Enter` для «=», `Backspace` для удаления, `Escape` для очистки).

## Запуск тестов

В текущей версии приложения автоматические тесты не предусмотрены. Всю функциональность можно проверить вручную, следуя сценариям из бизнес-требований (вычисление, обработка деления на ноль, очистка, история и т.д.).

## Примечания по безопасности

- **Локальная работа:** Приложение не отправляет никакие данные наружу. Все вычисления выполняются на клиентской стороне. История вычислений сохраняется исключительно в `localStorage` вашего браузера и не передаётся на сервер.
- **Валидация ввода:** Все пользовательские вводы (нажатия кнопок, ввод с клавиатуры) проходят строгую фильтрацию. Допускаются только цифры, десятичная точка и символы операций. Использование `eval()` или `new Function()` для вычислений исключено — применяется безопасный парсинг.
- **Защита от XSS:** При отображении данных (результаты, сообщения об ошибках, записи истории) используется только свойство `textContent`, что предотвращает внедрение HTML или JavaScript.
- **Content Security Policy (CSP):** Страница включает заголовок `Content-Security-Policy`, ограничивающий источники скриптов и стилей только локальными ресурсами. Для корректной работы звукового клика разрешён `unsafe-inline` для скриптов и стилей, что является безопасным для локального использования.
- **Рекомендация:** Для обеспечения максимальной безопасности при локальном использовании держите браузер обновлённым до последней версии.
