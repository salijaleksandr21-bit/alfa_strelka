Вот юз-кейсы (сценарии использования), составленные на основе предоставленных бизнес-требований и описания бизнес-процесса. Все сценарии соответствуют правилам безопасности и этики.

---

### Юз-кейс 1: Создание новой задачи

**ID:** UC-01  
**Связанные БТ:** БТ-01, БТ-02, БТ-07  
**Актор:** Пользователь  
**Предусловие:** Пользователь авторизован в системе (сессия активна). Интерфейс отображается в веб-браузере.

**Основной сценарий:**
1. Пользователь нажимает кнопку «Новая задача».
2. Система отображает форму с полями «Название» и «Описание».
3. Пользователь вводит название задачи (непустое) и, при необходимости, описание.
4. Пользователь нажимает кнопку «Создать».
5. Система выполняет валидацию: проверяет, что поле «Название» не пустое.
6. Система создаёт задачу со статусом «К выполнению» и отображает её на доске задач.

**Альтернативный сценарий (пустое название):**
- 5a. Если поле «Название» пустое, система отображает сообщение об ошибке: «Название задачи обязательно». Задача не создаётся.

**Постусловие:** Новая задача с уникальным идентификатором добавлена в систему. Данные задачи (название, описание, статус) сохраняются в локальном хранилище (если реализовано БТ-06).

**Безопасность:** Все вводимые данные проходят санитизацию перед отображением для предотвращения XSS-атак. Поле «Описание» не является обязательным, но при его заполнении также проверяется на вредоносный код.

### Юз-кейс 2: Изменение статуса задачи

**ID:** UC-02  
**Связанные БТ:** БТ-02, БТ-03, БТ-07  
**Актор:** Пользователь  
**Предусловие:** На доске отображается хотя бы одна задача.

**Основной сценарий:**
1. Пользователь выбирает задачу и выполняет одно из действий:
   - Перетаскивает карточку задачи в другую колонку (например, из «К выполнению» в «В работе»).
   - Нажимает кнопку смены статуса на карточке задачи.
2. Система проверяет, что новый статус является допустимым (один из трёх: «К выполнению», «В работе», «Выполнено»).
3. Система обновляет статус задачи и перемещает карточку в соответствующую колонку на доске.

**Постусловие:** Статус задачи изменён. Изменение сохраняется в локальном хранилище (если реализовано БТ-06).

**Безопасность:** Система не позволяет установить статус, не входящий в предопределённый список. Перетаскивание реализовано только через UI, без возможности манипуляции через URL или API.

### Юз-кейс 3: Удаление задачи

**ID:** UC-03  
**Связанные БТ:** БТ-04, БТ-07  
**Актор:** Пользователь  
**Предусловие:** На доске отображается хотя бы одна задача.

**Основной сценарий:**
1. Пользователь нажимает кнопку удаления на карточке задачи.
2. Система отображает диалоговое окно с запросом подтверждения: «Вы уверены, что хотите удалить задачу?» и кнопками «Удалить» и «Отмена».
3. Пользователь нажимает кнопку «Удалить».
4. Система удаляет задачу из списка и скрывает её карточку с доски.

**Альтернативный сценарий (отмена удаления):**
- 3a. Пользователь нажимает кнопку «Отмена» или закрывает диалоговое окно. Задача остаётся без изменений.

**Постусловие:** Задача удалена из системы. Данные о задаче очищаются из локального хранилища (если реализовано БТ-06).

**Безопасность:** Удаление выполняется только после явного подтверждения пользователем. Система не позволяет удалить задачу без подтверждения (например, через двойной клик или скрытые команды).

### Юз-кейс 4: Фильтрация задач по статусу (опционально)

**ID:** UC-04  
**Связанные БТ:** БТ-05, БТ-07  
**Актор:** Пользователь  
**Предусловие:** На доске отображается хотя бы одна задача.

**Основной сценарий:**
1. Пользователь выбирает статус из выпадающего списка фильтра (например, «Все», «К выполнению», «В работе», «Выполнено»).
2. Система отображает только те задачи, статус которых соответствует выбранному фильтру.
   - Если выбран «Все», отображаются все задачи без фильтрации.

**Постусловие:** Отображение задач обновлено в соответствии с выбранным фильтром. Фильтр не изменяет данные задач, а только влияет на их видимость.

**Безопасность:** Фильтрация выполняется на стороне клиента (или сервера с валидацией входных данных). Выпадающий список содержит только предопределённые значения, исключая возможность инъекции.
