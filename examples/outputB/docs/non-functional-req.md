Вот нефункциональные требования (НФТ), сформированные на основе предоставленных бизнес-требований, бизнес-процесса и с учётом правил безопасности и этики.

### Нефункциональные требования (НФТ)

| Категория | ID | Описание требования |
| :--- | :--- | :--- |
| **Безопасность** | НФТ-01 | **Валидация и санитизация ввода:** Все пользовательские вводы (название и описание задачи) должны проходить валидацию на стороне клиента и сервера (если применимо) для предотвращения XSS-атак и инъекций. Специальные символы и HTML-теги должны экранироваться или удаляться. |
| **Конфиденциальность** | НФТ-02 | **Локальное хранение данных:** Все данные задач, сохраняемые в локальном хранилище браузера (localStorage), должны быть зашифрованы. Личные данные пользователя (названия и описания задач) не должны покидать браузер пользователя без его явного согласия. |
| **Безопасные настройки по умолчанию** | НФТ-03 | **Сокрытие внутренней информации:** Система не должна раскрывать внутренние API-эндпоинты, версии библиотек или учётные данные в клиентском коде (JavaScript). Все конфиденциальные настройки должны быть вынесены на серверную часть или защищены. |
| **Производительность** | НФТ-04 | **Время отклика интерфейса:** Все операции пользователя (создание, перемещение, удаление, фильтрация задач) должны выполняться с задержкой не более 200 миллисекунд при стандартных условиях работы (до 1000 задач в системе). |
| **Надёжность** | НФТ-05 | **Целостность данных при сбоях:** В случае ошибки записи в локальное хранилище (например, превышение лимита памяти) система должна уведомить пользователя о невозможности сохранения данных, но не должна терять уже существующие задачи. |
| **Удобство использования (Usability)** | НФТ-06 | **Подтверждение необратимых действий:** Перед удалением задачи система должна запрашивать подтверждение у пользователя (как указано в бизнес-процессе). Отмена удаления должна возвращать пользователя к исходному состоянию без потери данных. |

**Примечание:** В рамках требований безопасности и конфиденциальности, опциональная функция `БТ-06` (сохранение данных между сессиями) реализована с обязательным шифрованием данных в локальном хранилище.
