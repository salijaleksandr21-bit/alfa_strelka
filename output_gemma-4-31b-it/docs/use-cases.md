Ниже представлены юз-кейсы, разработанные на основе предоставленных бизнес-требований и описания бизнес-процесса. Все сценарии спроектированы с учетом принципов безопасности: валидация ввода на стороне системы и отсутствие сбора персональных данных.

# Спецификация Use Cases: Конвертер валют

## UC-01: Конвертация суммы в другую валюту
**Описание:** Пользователь переводит сумму из одной валюты в другую, используя актуальный курс.
**Актор:** Пользователь.
**Связанные требования:** [БТ-01], [БТ-02], [БТ-03], [БТ-05], [БТ-07].

| Шаг | Действие Актора | Реакция Системы |
| :--- | :--- | :--- |
| 1 | Открывает страницу конвертера в веб-браузере. | Загружает интерфейс и запрашивает актуальные курсы из внешнего API. |
| 2 | Выбирает исходную валюту из списка (минимум 10 доступных). | Фиксирует выбор исходной валюты. |
| 3 | Выбирает целевую валюту из списка. | Фиксирует выбор целевой валюты. |
| 4 | Вводит числовое значение суммы. | **[БЕЗОПАСНОСТЬ]** Валидирует ввод на соответствие числовому формату. Рассчитывает результат на основе актуального курса и мгновенно отображает его на экране. |

---

## UC-02: Смена направления конвертации
**Описание:** Пользователь меняет местами исходную и целевую валюты для быстрого пересчета в обратную сторону.
**Актор:** Пользователь.
**Связанные требования:** [БТ-04].

| Шаг | Действие Актора | Реакция Системы |
| :--- | :--- | :--- |
| 1 | Нажимает кнопку смены направления «⇄». | Меняет значения в полях «Исходная валюта» и «Целевая валюта» местами. |
| 2 | — | Автоматически пересчитывает текущую введенную сумму по новому направлению и обновляет результат. |

---

## UC-03: Работа при недоступности внешнего API
**Описание:** Обеспечение работоспособности сервиса при сбоях на стороне поставщика данных.
**Актор:** Пользователь.
**Связанные требования:** [БТ-06].

| Шаг | Действие Актора | Реакция Системы |
| :--- | :--- | :--- |
| 1 | Открывает страницу или инициирует запрос курса. | Пытается установить соединение с API. При получении ошибки или таймаута переходит к использованию локального кеша. |
| 2 | Выбирает валюты и вводит сумму. | Выполняет расчет на основе последнего сохраненного курса. |
| 3 | — | Отображает результат конвертации и выводит предупреждение: «Курс от [дата]. Актуальные данные временно недоступны». |

## UC-04: Обработка некорректного ввода суммы
**Описание:** Предотвращение ошибок расчета при вводе недопустимых данных.
**Актор:** Пользователь.
**Связанные требования:** [БТ-03].

| Шаг | Действие Актора | Реакция Системы |
| :--- | :--- | :--- |
| 1 | Вводит в поле суммы отрицательное число, текст или спецсимволы. | **[БЕЗОПАСНОСТЬ]** Блокирует выполнение расчета, чтобы избежать системных ошибок или некорректных данных. |
| 2 | — | Выводит сообщение об ошибке: «Введите корректную сумму». Поле результата очищается или остается неизменным. |
