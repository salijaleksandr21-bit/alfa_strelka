На основе предоставленных бизнес-требований и описания процессов, я сформировал перечень нефункциональных требований (НФТ), сфокусированных на качестве, производительности и безопасности системы.

### Нефункциональные требования (НФТ)

| ID | Категория | Требование | Описание |
|:---|:---|:---|:---|
| **НФТ-01** | **Безопасность** | Валидация и санитизация ввода | Все пользовательские вводы (поля ввода суммы) должны проходить строгую валидацию на стороне клиента и сервера (если применимо) для предотвращения XSS-атак и инъекций. Недопустимо использование функций типа `eval()` для обработки ввода. |
| **НФТ-02** | **Конфиденциальность** | Защита локальных данных | Личные данные пользователя или история его конвертаций не должны покидать браузер без явного согласия. Любые данные, сохраняемые в `localStorage` или `IndexedDB` (например, кешированные курсы), должны храниться в зашифрованном виде или в формате, исключающем утечку чувствительной информации. |
| **НФТ-03** | **Безопасность** | Безопасные настройки по умолчанию | Система не должна раскрывать внутренние API-ключи, токены доступа к внешним сервисам курсов валют или структуру внутренних API в клиентском коде (JS-бандле). Все запросы к API должны проходить через защищенный прокси-слой. |
| **НФТ-04** | **Производительность** | Время отклика интерфейса | Результат конвертации (БТ-03) должен отображаться мгновенно (задержка не более 100 мс после ввода символа), чтобы обеспечить ощущение работы в реальном времени. |
| **НФТ-05** | **Надежность** | Отказоустойчивость (Graceful Degradation) | В случае недоступности внешнего API, система должна переключаться на кешированные данные без прерывания работы интерфейса и с четким уведомлением пользователя о дате актуальности данных (согласно БТ-06). |
| **НФТ-06** | **Совместимость** | Кроссбраузерность | Интерфейс должен корректно отображаться и функционировать в последних двух стабильных версиях основных браузеров (Chrome, Firefox, Safari, Edge). |

---
**Примечания эксперта по безопасности:**
*   В бизнес-процессе указано кеширование курсов. Я добавил требование **НФТ-02**, чтобы гарантировать, что механизм кеширования не станет вектором для хранения или утечки данных пользователя.
*   Поскольку система взаимодействует с внешним API, в **НФТ-03** зафиксирован запрет на хранение API-ключей в открытом виде на фронтенде, что является критической уязвимостью в подобных приложениях.
