Ниже представлены детальные функциональные требования (ФТ), разработанные на основе предоставленных бизнес-требований, юз-кейсов и фич. Каждое требование включает спецификацию безопасности в соответствии с заданными правилами.

---

## Функциональные требования (ФТ)

### ФТ-01: Инициализация интерфейса и защита заголовков
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-01 |
| **Источник** | UC-01, Features (Тёмная тема, Название «CurrFlow») |
| **Входные данные** | Запрос пользователя на открытие страницы (GET /). |
| **Ожидаемый результат** | Страница загружается с тёмной темой, заголовком «CurrFlow», и установленными заголовками безопасности (CSP, X-Frame-Options). |
| **Описание** | При загрузке страницы система должна отобразить интерфейс в тёмной теме на русском языке. Сервер должен передавать заголовки безопасности, запрещающие выполнение скриптов из сторонних источников и встраивание страницы в iframe. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: В исходных данных не было указано про CSP. Добавлено требование принудительного включения Content Security Policy (CSP) для предотвращения XSS атак. Запрещено выполнение пользовательского кода через `eval` или `innerHTML` без санитизации. |

### ФТ-02: Получение курсов валют из доверенного API
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-02 |
| **Источник** | БТ-02, Features (Обновление каждые 60 сек) |
| **Входные данные** | Таймер (60 сек), отсутствие локального кэша. |
| **Ожидаемый результат** | Система делает запрос к заранее определённому, доверенному эндпоинту API, получает JSON-ответ, валидирует его структуру и сохраняет в кэш. |
| **Описание** | Система автоматически запрашивает актуальные курсы валют каждые 60 секунд. URL API должен быть жёстко задан в конфигурации приложения (не может быть изменён пользователем). Ответ должен быть валидирован на соответствие ожидаемой схеме JSON. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Запрещено использование динамических URL из пользовательского ввода. Требуется валидация ответа API (проверка типов данных, наличие обязательных полей) перед использованием. Запрещено выполнение кода, полученного от API. |

| **Источник** | БТ-05, Features (11 валют) |

| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Список валют должен быть хардкодом на стороне клиента или загружаться из безопасного источника. Запрещено динамическое добавление валют на основе пользовательского ввода для предотвращения инъекций в логику выбора. |

### ФТ-04: Валидация ввода суммы
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-04 |
| **Источник** | БТ-03, Features (Только числа с точкой), UC-02 (Альт. поток 5a) |
| **Входные данные** | Текстовый ввод пользователя в поле «Сумма». |
| **Ожидаемый результат** | Поле принимает только числа (0-9) и одну точку. Отрицательные и нечисловые значения отклоняются с сообщением об ошибке. |
| **Описание** | При вводе система в реальном времени проверяет формат. Если введено не число, отрицательное значение или символы, отличные от цифр и точки, ввод блокируется или очищается. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Требуется строгая валидация на стороне клиента (regex) и повторная валидация перед вычислением. Санитизация ввода обязательна для предотвращения XSS при отображении ошибок. Максимальная длина ввода ограничена (например, 20 символов) для защиты от переполнения буфера. |

### ФТ-05: Расчет конвертации
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-05 |
| **Источник** | БТ-03, Features (Округление до 2 знаков) |
| **Входные данные** | Исходная сумма, коды валют, текущий курс. |
| **Ожидаемый результат** | Отображается результат конвертации, округленный до 2 знаков после запятой, и текущий курс (1 = X). |
| **Описание** | При изменении суммы или валют система мгновенно пересчитывает результат. Результат должен быть отформатирован с разделителем тысяч (опционально) и двумя знаками после запятой. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Все вычисления должны выполняться с использованием безопасных типов данных (например, Decimal), чтобы избежать ошибок округления. Вывод результата должен быть экранирован (HTML-кодирование) для предотвращения XSS, если результат может содержать спецсимволы. |

| :--- | :--- |
| **ID** | ФТ-06 |
| **Источник** | БТ-04, Features (Кнопка «⇄») |
| **Входные данные** | Нажатие кнопки «⇄». |

| **Описание** | Кнопка меняет местами выбранные валюты. Если сумма была введена, она сохраняется, но пересчитывается в новую целевую валюту. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Система должна убедиться, что после смены валюты обе валюты существуют в доверенном списке (ФТ-03). Это предотвращает использование несуществующих кодов валют в логике приложения. |

### ФТ-07: Обработка недоступности API (Кэширование)
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-07 |
| **Источник** | БТ-06, UC-04, Features (Жёлтая плашка) |
| **Входные данные** | Ошибка соединения с API или тайм-аут. |
| **Ожидаемый результат** | Система использует данные из локального кэша, отображает результат и жёлтую плашку «Данные устарели» с датой последнего обновления. |
| **Описание** | Если API недоступен, система не блокирует работу, а предлагает использовать сохранённые ранее курсы. Пользователь явно уведомляется о статусе данных. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: Кэш должен содержать метку времени. Если кэш старше определенного порога (например, 24 часа), система должна принудительно запрашивать данные или блокировать конвертацию. Это предотвращает использование устаревших данных, которые могут быть скомпрометированы или неактуальны. |

### ФТ-08: Сохранение состояния в LocalStorage
| Параметр | Описание |
| :--- | :--- |
| **ID** | ФТ-08 |
| **Источник** | Features (Сохранять валюты и сумму) |
| **Входные данные** | Выбранные валюты, введенная сумма. |
| **Ожидаемый результат** | Данные сохраняются в `localStorage` и восстанавливаются при следующем входе. |
| **Описание** | Для удобства пользователя последние настройки сохраняются локально в браузере. |
| **Примечания по безопасности** | **[ИЗМЕНЕНО ДЛЯ БЕЗОПАСНОСТИ]**: В `localStorage` **запрещено** хранить чувствительные данные (пароли, токены, PII). Разрешено хранить только коды валют и числовое значение суммы. Данные должны быть очищены при явном сбросе настроек. |

## Заключение
Все функциональные требования переработаны с учетом принципов безопасности по умолчанию (Security by Design). Критические изменения, такие как внедрение CSP, ограничение доступа к API и запрет на хранение чувствительных данных в LocalStorage, зафиксированы как обязательные. Система не требует аутентификации, что снижает поверхность атаки, но требует строгой защиты от XSS и CSRF (на уровне заголовков и логики).
